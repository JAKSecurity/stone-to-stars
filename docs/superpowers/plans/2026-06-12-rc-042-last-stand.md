# RC-042 The Last Stand Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development or superpowers:executing-plans.

**Goal:** The C4 finale per spec §B2 — `docs/superpowers/specs/2026-06-12-c4-space-age-last-stand-design.md`: invader formation waves + mothership in a fixed arena, victory screen, `lastStandWon` flag, replayable, defeat = normal banking.

**Architecture:** all wave/formation/phase math is pure in a new `src/run/invasion.ts`; the finale rides the EXISTING expedition pipeline via a special biome + `finale` flag so RunScene's init/biome/walls/banking machinery is reused, with a formation controller layered on top. Victory is a new DOM screen beside `runEndScreen`.

## Pinned cross-module contracts (both dispatches MUST honor these exactly)

- `BiomeDef` gains optional `finale?: boolean`. New biome `last_stand` (id, `minAge: 'space'`, `requiresTech: 'planetary_defense'`, `finale: true`, dark "last city" palette, NO obstacles, spawnTable `{ invader_drone: 1 }` — present only so apex/danger helpers stay total; the formation system ignores it).
- `availableExpeditions` (src/run/expedition.ts) SKIPS `finale` biomes — the finale never appears as a normal card. New export `lastStandUnlocked(civ): boolean` = `isResearched(civ, 'planetary_defense')`.
- `Expedition` type gains optional `finale?: boolean`; the expedition screen launches the finale as `{ biomeId: 'last_stand', tier: 8, finale: true }` with NO mutators.
- `src/run/invasion.ts` exports:
  - `interface InvasionWave { rows: string[]; cols: number; marchSpeed: number; dropPx: number; spacing: number }` (`rows` = enemy id per row, top to bottom)
  - `WAVES: InvasionWave[]` — 5 waves, escalating (drone-only → mixed → elite rows; speeds/drops rise)
  - `interface FormationState { x: number; dir: 1 | -1 }`
  - `formationStep(s: FormationState, dtMs: number, marchSpeed: number, minX: number, maxX: number): { s: FormationState; dropped: boolean }` — pure: marches x by dir·speed·dt, on crossing a bound clamps, reverses dir, reports `dropped: true` (caller applies dropPx toward the player).
  - `MOTHERSHIP_HP_MULT` (vs. tier-8 boss base), `mothershipPhase(hpFrac: number): 1 | 2 | 3` (thirds: >2/3→1, >1/3→2, else 3).
- New enemy defs in `src/run/enemyData.ts`: `invader_drone`, `invader_soldier`, `invader_elite`, `invader_mothership` — space-tier stats, RC-040 attack profiles (drone none, soldier beam, elite mortar, mothership per-phase in RunScene). New sprites (classic invader silhouettes + a wide mothership) in `src/art/sprites/enemies.ts`, registered. Regular biome spawn tables are NOT touched (no leak by construction).
- `RunResult` gains `finaleVictory?: boolean`; `CivState` gains `lastStandWon?: boolean`; `applyRunResult` sets it true when `result.finaleVictory` (never unsets).
- RunScene init: `RunInit.expedition.finale` routes to finale mode (formation spawning instead of the normal spawner/dungeon/POI systems; walls on; no deposits/POIs/dungeon/mutators; drafts/relics/gems work normally).
- Victory: mothership death → gem vacuum + fanfare → `finish(false)` with `finaleVictory: true` in `pendingComplete`. main.ts `onRunComplete` routes `finaleVictory` results to `renderVictoryScreen` (new `src/ui/victoryScreen.ts` + a `#victory` container following `index.html`'s existing screen-div pattern) instead of the run-end screen; its Continue button returns to the civ screen. Civ screen header shows 🏆 when `civ.lastStandWon`.

### Dispatch 2 — RunScene finale mode

**Files:** modify `src/scenes/RunScene.ts` (sequential after Dispatch 1; no other file).

- `finale` flag from init: skip normal enemy spawner, POI rolls, dungeon population, deposits; arena = walled rectangular layout (reuse RC-038 wall containment; biome palette supplies the floor).
- Formation controller: per wave, spawn the grid via the existing `spawnEnemyAt` (awake, no despawn timers), track members + a `FormationState`; each update, `formationStep(...)` moves the ORIGIN and members keep grid offsets (set body velocity or position directly — members remain normal physics bodies so weapons/contact work); on `dropped`, shift the formation origin `dropPx` toward the player side. Members killed leave the formation set; when empty → brief inter-wave beat (banner + 1.5s) → next wave. Waves 1–5 from `WAVES`.
- Wave 6: spawn `invader_mothership` (isBoss, boss HP bar, HP = tier-8 boss base × `MOTHERSHIP_HP_MULT`) anchored to the arena top, slow horizontal drift; per `mothershipPhase(hpFrac)`: phase 1 mortar-profile plasma volleys, phase 2 beam-profile sweeps + periodic `invader_drone` spawns (cap concurrent), phase 3 both faster (reuse the RC-040 enrage tint/speed conventions).
- Mothership death → victory: suppress the normal startCeremony path for the finale (guard `this.finale`), run gem vacuum + victory fanfare (reuse zone-cleared cue), `finish(false)` with `finaleVictory: true` added to `pendingComplete`. Player death anywhere → existing `finish(true)` (banking unchanged; `finaleVictory` absent).
- HUD: wave counter line during the finale (`Wave 2/5` / `MOTHERSHIP`).
- Gates: tsc + full vitest + build. One commit: `feat(RC-042): finale mode in RunScene - formation waves, mothership, victory`.
